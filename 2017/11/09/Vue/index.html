VUE框架学习之SPA

发表于 2017-11-09

VUE介绍

Vue.js（读音 /vjuː/，类似于 view 的读音）是一套构建用户界面(user interface)的渐进式框架。与其他重量级框架不同的是，Vue 从根本上采用最小成本、渐进增量(incrementally adoptable)的设计。Vue 的核心库只专注于视图层，并且很容易与其他第三方库或现有项目集成。另一方面，当与单文件组件和 Vue 生态系统支持的库结合使用时，Vue 也完全能够为复杂的单页应用程序提供有力驱动。

实践步伐

由于自己原生js功底很弱，所以对于框架的学习也有点畏惧，迟迟不肯行动。今天试着用VUE实现了一个单页面应用，单页面应用的基本元素：页面路由，通过某种方式，如 URL hash 来说明表明当前所在的页面，并拥有从一个页面跳转到另外一个页面的入口。模块化很强，减少了普通页面切换的时间，很方便，也很迅速。

实现一个简单的单页面应用可以通过vue-cli及其模板构造出一个大致的页面。也就是
$ npm install -g vue-cli $ vue init webpack-simple my-project $ cd my-project $ npm install $ npm run dev
其中：
npm run dev: Webpack + vue-loader with proper config for source maps & hot-reload.(在电脑的8080端口可以查看界面) npm run build: build with HTML/CSS/JS minification. (有个dist文件夹，文件夹内部全是编译过的文件，也就是打包出来的文件，通过index.html可以直接显示界面)
各文件夹大概内容以及其作用如下：
build文件夹内一般是一些config文件。
src文件夹里是我们用到的，内部文件可用的大概有以下几个：
APP.vue是整体界面描述，里面包含有很多组件及路由。
router文件夹里index.js存放着对应的路径及其组件声明。
main.js指明入口文件。
components里一般放置组件的.vue文件。

基本语句

<router-link to="home">Home</router-link>
表示目标路由的链接。当被点击后，内部会立刻把 to 的值传到 router.push()，所以这个值可以是一个字符串或者是描述目标对象。
<router-link to="home">Home</router-link>
也就是Home
<router-link v-bind:to="'home'">Home</router-link>
<router-link :to="'home'">Home</router-link>

对于router里index.js大致格式如下：
{path: '/', redirect: '/HelloWorld'}, {path: '/HelloWorld', component: HelloWorld}, {path: '/HelloVue', component: HelloVue}

export default { name: 'HelloWorld' } export default 提供了一个接口，可以通过import来进行引用。

对于.vue模板内必须有一个最外部的div来包含其他所有东西，也只能有一个最外部div.

遇到的问题

1 npm run dev时报错，很多跟格式有关的错误，比如多空格
解决方法：EsLint帮助我们检查Javascript编程时的语法错误,禁用Eslink.通过更改webpack的配置文件。
2 npm run dev页面找不到
解决方法：有可能是因为端口被占用。
